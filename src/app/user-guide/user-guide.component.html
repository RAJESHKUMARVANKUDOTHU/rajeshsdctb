Social Distancing User manual

ソーシャル・ディスタンシング ユーザーガイド

1. LOGIN PAGE

  • Open a browser and search for https://sdctb.sensegiz.com/login. The user will be directed to the login page where the user has to enter the credentials provided to them.

1. ログインページ

  • ブラウザを開き、https://sdctb.sensegiz.com/loginを検索します。ユーザーはログインページに移動し、ID及びパスワードを入力する必要があります。

2. HOME PAGE

  • Once the user has logged in the landing page which appears is the home page. The homepage displays an overall statistical data of all registered users.
    1. Registered users list – left pane
    2. There are 4 donuts shows number of Active, Registered, Offline and Infected Users.
    3. Trend of ‘number of contacts made over last 10 days’
    4. Proximity and violation alert settings
    5. Frequent violators of the day
    6. Cumulative time
  • Registered users show the total number of registered users on the portal. Active users show number of devices which are sending data from last 24 hours. Offline users are the users which are registered but not active since last 24hours. Infected users are the users among registered users who have marked as infected.
  • In registered user left pane, the admin gets the report of any particular user by selecting the name of the user that is available in the list.
  • Color coding in the registered users’ left pane is as follows
    1. Green: The last sync is less than aday.
    2. Yellow:Thelast sync is within 1-2 days.
    3. Orange orRed:The last sync is more than 2 days
  • The admin can also see the proximity selected by for them. The admin can also see the violation alert i.e. the selected threshold allowed for the maximum violation per person per day.
  • The Frequent violators section displays the user with the maximum number of interactions they had with other users on that particular day.If a user crosses the maximum limit set in the Automatic alert,the admin will get an immediatealert. Additionally, a provision has been given to the admin to send a warning to the individual user if they click on the sendwarning button.
  • The Number of contacts Graph will display the day-wise graphical data which represents the total number of social distancing violations that happened for the past 10 days. The admin can also take a print out of the report or can save it as a JPEG orPNG.
  • The users who violate Social Distancing for a maximum duration with other users even after getting an alert are displayed under Maximum contact time.

2. ホームページ

  • ユーザーがログインするとホームページに移動します。ホームページには、登録されているすべてのユーザーの全体的な統計データが表示されます。
    1. 登録ユーザーリスト–左サイド
    2. 4つの円グラフがあり、アクティブ、登録済み、オフライン、および感染したユーザー数を示しています。
    3. 「過去10日間の接触機会の回数」
    4. 接触距離（ソーシャルディスタンス）および接近警告の設定
    5. その日に頻繁に接触を繰り返した人のリスト
    6. 各ユーザーの接触累積時間
  • 登録ユーザーとはポータルに登録されているユーザー総数です。アクティブユーザーとは過去24時間内にデータを送信しているデバイス数です。オフラインユーザーとは登録されているが過去24時間以上アクティブではないユーザー数です。感染ユーザーとはイベント後登録ユーザーの中でコロナ感染が判明されたユーザーです。
  • 管理者は左サイドにある登録ユーザーリストから個人ユーザーの名前を選択することにより、特定個人ユーザーのレポートを取得できます。
  • 左サイドにある登録者リストの色分けは次のとおりです。
    1. 緑: 最新のアップデートされたデータが1日未満の方です。
    2. 黄色：最新のアップデートされたデータが1〜2日以内の方です。
    3. オレンジまたは赤：最新のアップデートされたデータが2日以上の方です。
  • 管理者は選択した許容接触距離（ソーシャルディスタンス制限）のしきい値情報を確認できます。また、管理者は違反アラート、すなわち、各ユーザーの一日当たりの最大接触回数のしきい値も確認することができます。
  • 「頻繁な違反者」の欄には、指定された日の各個人の接触回数が多い人がリストされます。設定された最大接触回数制限値を超えると、E-mailにて管理者にアラートが送信されます。さらに、管理者は[警告送信]ボタンをクリックすることにより、頻繁に接触するユーザーに対し接触回数が多いことをしらせる警告を発報することが出来ます。
  • 「接触した回数」のグラフには、過去10日間に発生した一日当たりの接触合計回数をデイリーごとに表示しています。管理者は、レポートを印刷したり、JPEGまたはPNGとして保存したりすることもできます。
  • アラートを受け取った後でも、他のユーザーとの最大期間で社会距離拡大に違反したユーザーは、各ユーザーの接触累積時間で表示されます。

3. MANAGE PEOPLE

    In Manage people tab there are three sections

  1. Manage Finds
    • The admin can view the details of all the registered users and the FIND ID that is assigned to the user
    • Most importantly, the admin will be able to mark a user as infected if he/she is found to be infected after a few days.
    • The admin can also reuse/reassign the same FIND for a different person by selecting the edit option and update the details of the user. Admin can reassign the devices one at a time.
    • The admin can update the shift details of a particular user by selecting the ‘shift dropdown’ box.
    • The admin can edit the existing information associated with the FIND like device ID, device name, employee ID, Shift, email ID and mobile number.
    • Using the Add FINDs option, the admin will be able to add a new FIND to the existing system and assign it to a user.
    • Moreover, to assign FIND tags to multiple people, admin can fill up out standard template format and upload it using ‘upload file’ option at left top corner.
    • The admin can use the search option to view,edit or delete any particular user.
    • The admin can also view the FIND battery status of each user.Color coding for battery status is
      1. Green:High
      2. Yellow:Medium
      3. Red:Low
  2. Manage Gateways
    • The admin can view the registered Gateways, edit them and can also add new Gateway using the Add Gateways option.
    • The admin can see multiple Gateways with their firmware version. Moreover, we are soon going to add the BLE version section for each gateway.
    • The search option can be used to find details of any particular Gateway.
  3. Manage Users.
    • The admin can view or edit the details of the Authorized people who will receive social distancing violators and inactivity alerts.
    • The admin can also add new users to receive the alert using the Add Users option. For example, if the admin does not wish to receive all the email alerts then the person who is added in the manage user tab will receive all the alerts.

3. 管理画面

    管理画面タブには、3つのセクションがあります。

  1. タグデバイス（FINDS）の登録・管理
    • 管理者は、登録されているすべてのユーザー情報と、ユーザーに割り当てられているタグデバイス(FIND)IDを確認できます。
    • 最も重要なことは、あるユーザーが数日後にウイルス感染していることが判明した場合、管理者はその登録ユーザーリストからCOVID-19感染者として特定マーク出来ることです。
    • 管理者はタグデバイスのユーザー情報を書き換えることにより、同じタグデバイスでも他のユーザーに再利用することも可能です。
    • 管理者は、ユーザーが使用するタグデバイスの時間帯（シフト）をドロップボックスから変更することができます。
    • 管理者は、デバイスID、デバイス名、従業員ID、シフト、電子メールアドレス、携帯電話番号などを編集できます。
    • タグデバイスの追加オプションを使うことで、管理者は新しいタグデバイスFINDを既存のシステムに追加することができます。
    • さらに、複数のタグデバイスを登録するためには、管理者は標準テンプレートに登録情報入力し、画面左上の[ファイルのアップロード]オプションをクリックしてアップロードすることできます。
    • 管理者は、検索オプションを使用して、特定のユーザーを表示、編集、または削除できます。
    • 管理者は、各ユーザーのタグデバイス（FIND）のバッテリー残量を確認することもできます。バッテリー残量の色分けは；
      1. 緑:高
      2. .黄色:中
      3. 赤:低
  2. ゲートウエイの登録・管理
    • 管理者は、登録済みのゲートウェイを表示、編集したり、[ゲートウェイの追加]オプションを使用して新しいゲートウェイを追加したりできます。
    • 管理者は、ファームウェアバージョンとともに複数のゲートウェイを表示できます。さらに、ゲートウェイごとにBLEバージョンセクションをまもなく追加する予定です。
    • 検索オプションを使用して、特定のゲートウェイの詳細を見つけることができます。
  3. ユーザーの登録・管理
    • 管理者はソーシャルディスタンス内に接近した違反者の詳細情報入手、または編集ができます。
    • 管理者は、[ユーザーの追加]オプションを使用して、アラートを受信する新しいユーザーを追加することもできます。たとえば、管理者がすべての電子メールアラートを受信したくない場合は、[ユーザーの管理]タブに追加された人がすべてのアラートを受信することもできます。

4. SOCIAL DISTANCING

  • In social distancing tab admin can see the live data with the name of base person, contact person, violation start time, violation end time and total time. Violation start time and end time column has DDMMYY HHMMSS format of 12 hours. This time is visible according to the country’s standard time. In India this time is visible in IST
  • Admin can select how many data should be visible in this display using the pagination option in the dropdown.
  • In the live data, there will be two different violations if the base person and contact person has interchanged. Whereas, multiple violation with the exact same base person and contact person within the 5mins duration will get merged.

4. ソーシャル・ディスタンシング

  • ソーシャルディスタンスタブでは、管理者はベースパーソン、コンタクトパーソン、接触（違反）開始時間、接触（違反）終了時間、接触合計時間のライブデータを見ることが出来ます。違反の開始時刻と終了時刻の表示方法は各国の標準時間に変更可能です。今回は日本標準時間で表示されています。
  • 管理者は、ページのドロップダウンを使用して表示されるすべてのデータを見ることが出来ます。

5. SETTINGS

  • Settings tab allows admin to set and/or edit the various parameters as follows,
    1. Violation limit
    2. Two step authentication
    3. Alert configuration
    4. Proximity distance and wearable type
    5. Upload company logo
  • Apart from above there are two parameter which admin can not select without informing to the SenseGiz support team and they are as follows,
    1. Define Buffer
    2. Shift time
  • Define Buffer: This is the threshold which admin can set for all the FINDs registered. Once FIND will store these many data packets only after that the data will get send to the Gateway. This option helps to increase the FIND battery back-up but, it is not recommended as it can cause the data delay. That is why we have kept this setting access with super-admin.
  • Define Violation Limit:Using this option, the admin can set the proximity threshold in meters. This will be applicable for all the FINDs which are registered. The admin can select the proximity among 1m, 2m and 3m. For setting the customized proximity admin has to contact the SenseGiz team.
  • Two step Authentication:This setting allows an admin to opt for two step authentication and avail more safety to the account. If any other person apart from admin tries to log in then the admin will get an OTP to the registered mobile number and email address. Only after sharing this OTP the portal will accept and allow the login.
  • Alert Configuration:This setting allows an admin to set the alert type. There are 4 different types of alert, once you have selected the alert accordingly FIND devices will indicate when the violation happens.
    • Buzzer followed by LED for every new person: On the violation buzzer will beep for initial 10 seconds followed by LED blinking for rest of the time till violation ends. This alert will work only once if the same pair is violating again and again.
    • Only LED: LED on the FIND devices will blink for entire violation time.
    • Only Buzzer: Buzzer on the FIND devices will beep for entire violation time.
    • Buzzer followed by LED blinking for every new violation:On the violation buzzer will beep for initial 10 seconds followed by LED blinking for rest of the time till violation ends. This alert will work I every time even if the same pair is violating again and again.
    • Buzzer followed by LED after set periodic interval: On the violation buzzer will beep for initial 10 seconds followed by LED blinking for 50 seconds and the cycle repeat after the set periodic interval.

  • Define Proximity and Select the wearable type: Using this option admin can select the proximity threshold in meters as well as there is an option to select which FIND version you are using; a wrist band or the pendant.
  • Define Shift Time:This option is for setting the shifts. As of now only super-admin that means SenseGiz support team can set the threshold for the admin. Admin can select,
    • 24 hours if they are working in all 3 shifts
    • 16 hours for 2 shifts
    • 8 hours for 1shift
    Moreover, the hours window is customizable. This will save the FIND battery life tremendously. Lower the shift time more the FIND battery lasts. In future one company can also operate in multiple shifts and this feature will get ready soon.
  • Upload Logo:Using this option admin can upload their company logo in place of the SenseGiz logo.

5. 設定

  • [設定]タブでは、管理者は次のようにさまざまなパラメータを設定および/または編集できます。
    1. 一日当たりの最大接触制限回数
    2. 二段階認証
    3. アラートの構成
    4. ソーシャルディスタンス距離の設定とタグデバイスの種類
    5. 会社のロゴをアップロード出来ます
  • 上記とは別に、インディゴサポートチームに通知せずに管理者が選択できない2つのパラメーターがあります。それらは次のとおりです
    1. タグデバイス内に保存される接触データ量
    2. シフト時間
  • タグデバイス内に保存される接触データ量：これは、管理者が登録するすべてのタグデバイスに対して設定できるデータ保存量です。タグデバイスが多くのデータパケットを保存した後にゲートウェイに転送されます。この場合タグデバイスのバッテリー消費を節約するのに役立ちますが、サーバーへのデータ転送遅延を引き起こす可能性があるためお勧めしません。そのため、IndyGo社がこの設定へのアクセスを管理しています。
  • 最大接触制限回数：このオプションでは、管理者はソーシャルディスタンス制限値をメートル単位で設定できます。これは、登録されているすべてのタグデバイスに適用されます。管理者は、1m、2m、3mの中から選択できます。ソーシャルディスタンス制限値をカスタマイズしたい場合はインディゴチームに連絡する必要があります。
  • 二段階認証：二段階認証を選択することによりアカウントの安全性を高めることができます。管理者は登録された電子メールアドレスへOne Time Password（OTP）が送られてきます。もし、管理者以外の誰かが本ポータルにログインした場合、管理者の電子メールアドレスにOTPが送られてくるため不正ログインを拒むことが出来ます。
  • アラートの構成：管理者はアラートタイプを設定できます。アラートには4つの異なるアラート方法があります。各種アラートを選択することにより、それに応じたタグデバイスの接触違反表示方法が設定されます。
    • タグデバイスを持っている人に初めて設定距離範囲内に接近した時の警告：初めて接触する人にはブザー警告音が鳴り、そのあとLEDが点滅します。：ブザーは10秒間ビープ音を鳴らし、その後、設定距離内（違反）から離れるまでLEDは点滅し続けます。このアラートは、同じペアで初めて設定距離内に接近（違反）した時のみ機能します。
    • LEDのみ警告：タグデバイスFINDのLEDは、設定距離以内（違反）時間全体にわたって点滅します。
    • ブザーのみ警告：タグデバイスFINDのブザーは設定距離以内（違反）時間全体にわたってビープ音を鳴らします。
    • タグデバイスを持っている人に設定距離範囲内に接近した時の警告：初めての接触のみとは関係なく、同じペアで何度も接触する場合，毎回ブザー警告音が鳴り、そのあとLEDが点滅します。ブザーは10秒間ビープ音を鳴らし、その後、設定距離内（違反）から離れるまでLEDは点滅し続けます
    • 設定された時間間隔でブザーとLEDで警告：設定距離内（違反）に接近した場合、ブザーが最初の10秒間ビープ音を鳴らし、続いてLEDが50秒間点滅します。設定された時間間隔の後、繰り返し同じ警告内容が行われます。

  • 許容接近距離（ソーシャルディスタンス）とウェアラブルタイプの選択：管理者はメートル単位で許容接近距離（ソーシャルディスタンス）を選択できます。また、使用するタグデバイスFINDの種類を選択するオプションもあります。タグデバイスの種類はリストバンドタイプ、もしくはペンダントタイプです。
  • 接触監視機能の時間設定：このオプションは基本的には勤務時間シフトの情報を使うもので、勤務時間帯のみ監視機能を駆動させることによりタグデバイスの電池消費等の節約を行うものです。現在のところ、インディゴサポートチームのみが本設定の管理を行っています。
    • 3シフトすべてで働いている場合は24時間
    • 2シフトで16時間
    • 1シフトで8時間
    時間枠でカスタマイズ可能です。これにより、タグデバイスのバッテリー寿命が大幅に節約されます。将来的には、一つの会社内で複数のシフトを運営できる監視システムを持てるように、現在改善作業中です。
  • ロゴのアップロード：このオプションを使用すると、管理者はIndyGoロゴの代わりに自社のロゴをアップロードできます。

6. Report

  • Report tab allows admin to view and download reports. There are four different reports available for SDCT option B,
  • Admin can view and download the reports from last week or two weeks just by selecting the radio button. Moreover, if you want to view or download the last month report or for any duration then please do select the date range.
    1. Generic Report
      • The generic report shows violation details such as base person and contact person name with violation start time and end time with the total time in 12 hours format w.r.to the country time
    2. Cumulative Report
      • Cumulative report is specifically for the top management professionals, when CxO wants to see the crisp report of all the registered users with their total number of violations and violation duration.
      • Following report shows total number of violations and respective total time for all the 51 registered users
    3. Social Distancing Violator’s Report based on Username
      • This report gives the detailed data for a particular violator and his primary contacts with the contact time as well as total time for each violations the violator has made.
    4. Summary Report of an Infected User
      • This report shows the summary of primary contacts for the selected infected person and for selected duration. Once admin marks someone as covid-19 infected then this report gives idea that to whom all he met post marking as infected.
      • With our upcoming feature of isolation and send warning, an automatic warning will be sent via email ‘To’ all the primary contacts keeping admin in the ‘CC’. Moreover, the admin can mark these primary contacts as isolated and accordingly take the precautionary measures.

6. レポート

  • [レポート]タブでは、管理者はレポートの表示、およびダウンロードができます。4種類のレポートがあります。
  • 管理者は選択ボタンから前週1週間分、または2週間分のレポートを表示、もしくはダウンロードできます。さらに、過去のデータに関しては、日付範囲を選択することにより任意の期間のレポートを表示またはダウンロードすることが可能です。
    1. 全体の接触機会レポート
      • 全体の接触機会レポートは、ベースパーソンとソーシャルディスタンス内に接触した人の名前、接近開始時刻と終了時刻、及びその接触累積時間が表示されます。
    2. 累積された接触機会レポート
      • 累積された接触機会レポートは、管理者が指定した期間内でのすべての登録ユーザーに対する接触した合計数と、接触累積トータル時間を確認するためのものです。
      • 次のレポートは、51人の登録ユーザーすべての接触回数と接触累積時間を示しています。
    3. 特定された人のソーシャルディスタンス内接触者リスト
      • このレポートは、特定した人がソーシャルディスタンス内に接触した人の名前と、その接触開始時間、及び接触合計時間を示したものです。
    4. 感染者による濃厚接触者リスト
      • このレポートは、イベントが行われた後、COVID-19感染が判明した人から感染の連絡を受け、対象感染者がイベント期間中にソーシャルディスタンス内に近づいたすべての接触候補者をリスト化したものです
      • 管理者は、感染者と十分な接触があった候補者に対して、「イベント期間中に感染者と濃厚接触があった可能性がある」旨の警告メールを接触候補者に送信することにより、感染有・無の早期検査の実施、及び感染拡大を防ぐ対処方法が可能となります。
